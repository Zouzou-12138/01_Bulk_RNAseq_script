# Standard Bulk RNA-Seq Analysis Pipeline

## 1. Project Overview

This project is an automated Bulk RNA-seq analysis pipeline built on **Snakemake**. It is specifically optimized for mouse (**Mus musculus, GRCm39**) transcriptome data processing and follows industry-standard bioinformatics protocols.

**Core Features:**

* **Declarative Workflow**: Managed by Snakemake, supporting goal-driven execution and error recovery.
* **Standardized Quality Control**: Integrated FastQC and Cutadapt filtering (Q20, length > 75bp).
* **Novel Transcript Prediction**: De novo transcript assembly using StringTie.
* **Structured Output**: Automatically generates a 00-13 numbered directory system, consistent with professional bioinformatics reporting standards.

---

## 2. Directory Structure

The script automatically creates the following directory hierarchy during execution:

* **00_Rawdata**: Input directory for raw paired-end fastq files (`.fq.gz` format).
* **01_QC**: FastQC quality control reports and Cutadapt-trimmed data.
* **02_Mapping**: HISAT2 alignment results (sorted BAM files and indices).
* **03_AS**: Placeholder for Alternative Splicing analysis.
* **04_Novel**: StringTie transcript assembly and new isoform GTF files.
* **05_SNV**: Placeholder for SNV/Indel analysis.
* **06_GeneExpression**: Raw counts matrix generated by HTSeq-count.
* **07-13**: Pre-created directories for downstream analysis (DE, GO, KEGG, PCA, GSEA, etc.).

## 5. Data Preparation

### 5.1 Copy Raw Data

**Copy your raw sequencing data into the `00_Rawdata` folder** under the script directory (the script will create it if it does not exist):

```bash
# Ensure you are in the script directory
cd /path/to/01_Bulk_RNAseq_script

# Copy raw data to 00_Rawdata directory
cp /path/to/your/rawdata/*.fq.gz ./00_Rawdata/

```

### 5.2 File Naming Convention

Raw fastq files must follow these naming rules:

* Sample IDs must match those used in subsequent analyses.
* Format: `{SampleID}_L1_1.fq.gz` and `{SampleID}_L1_2.fq.gz`.

**Example:**

* `WT-testis-1_L1_1.fq.gz` / `WT-testis-1_L1_2.fq.gz`
* `C2-KO-testis-1_L1_1.fq.gz` / `C2-KO-testis-1_L1_2.fq.gz`

---

## 6. Configuration (config.yml)

Before running, modify the absolute paths in `config.yml` according to your server environment:

* **threads**: CPU core allocation (Default: 36).
* **ref_genome**: Path prefix for HISAT2 genome index.
* **gtf**: Path to the reference annotation file (e.g., `GRCm39.110.gtf`).
* **adapter**: R1 and R2 adapter sequences for trimming.

## 7. Running the Pipeline

### 7.1 Method 1: Using Python Wrapper Script (Recommended)

```bash
# Ensure you are in the script directory
cd /path/to/01_Bulk_RNAseq_script

# Run the full pipeline in the background (automatically initializes directory structure)
nohup python run_pipeline.py > pipeline.log 2>&1 &

```

### 7.2 Method 2: Direct Snakemake Execution (Flexible)

```bash
# Dry run to check logic
snakemake -n

# Formal execution
snakemake --cores 36 --printshellcmds

```

### 7.3 Monitoring Status

```bash
# View execution logs
tail -f pipeline.log

# Check active processes
ps -ef | grep snakemake

```

## 8. Analysis Methodology

1. **Quality Control**: Initial assessment with **FastQC**. **Cutadapt** is used to trim adapters and low-quality bases (Q20). Reads with a trimmed length < 75bp are discarded.
2. **Alignment**: Reads are aligned to the GRCm39 genome using **HISAT2** with the `--dta` flag enabled for downstream assembly.
3. **Transcript Assembly**: **StringTie** reconstructs transcripts guided by the reference GTF to identify novel isoforms.
4. **Quantification**: **HTSeq-count** calculates gene-level raw counts based on exon features for differential expression analysis.

## 9. Downstream Analysis

### 9.1 Create Sample Grouping Information

```csv

WT-testis-1,WT

C2-KO-testis-2,C2_KO
C26-DKO-testis-1,C26_DKO
C26-DKO-testis-2,C26_DKO

```

### 9.2 Execute Downstream Scripts

```bash
# Run using default R environment
nohup Rscript downstream_analysis.R > downstream.log 2>&1 &

```

### 9.3 Outputs

The following results will be generated in the script directory:

* **07_AdvancedQC**: Correlation heatmaps, expression heatmaps, and PCA plots.
* **08_DE**: Differential Expression Gene (DEG) lists and Volcano plots.
* **09_GO / 10_KEGG**: Enrichment analysis tables and visualizations.

## 10. Troubleshooting

| Issue | Likely Cause | Solution |
| --- | --- | --- |
| Input files not found | Path error or naming mismatch | Check `00_Rawdata` and naming conventions |
| Low alignment rate | Residual adapters or wrong index | Verify `config.yml` adapter sequences |
| Downstream error | Group file mismatch | Check `sample_group.csv` sample names |
| Out of memory | High concurrency on large data | Reduce `--cores` or increase RAM |
